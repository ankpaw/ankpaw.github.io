## 2026-04-08 - Added Keyboard Accessibility to Experience Timeline Cards
**Learning:** Custom interactive elements designed as cards (like the experience timeline accordions) often lack keyboard support (`tabIndex`, `role`, and `onKeyDown`) out of the box, rendering them unusable for keyboard and screen reader users. Simply adding an `onClick` handler to a `<div>` or custom UI card is insufficient for a11y.
**Action:** Always ensure that custom interactive elements acting as buttons have `role="button"`, `tabIndex={0}`, keyboard event handlers (for `Enter` and `Space`), and proper `aria-expanded`/`aria-controls` attributes if they toggle content. Ensure focus states are clear using `focus-visible`. Update click handler types (e.g. from `React.MouseEvent` to `React.SyntheticEvent`) to safely handle both mouse and keyboard inputs.
## 2026-05-04 - Added ARIA Label and Title to State-Driven Like Button
**Learning:** State-driven icon buttons (like "Like" or "Toggle" buttons) can be confusing for screen reader users if they only rely on visual cues (like color changes) and raw text (like a like count). The inner text does not explain the action, and decorative icons might be announced confusingly. Furthermore, disabled buttons without explanation frustrate users.
**Action:** For state-driven icon buttons, always add `aria-pressed={isToggled}`. Use a comprehensive `aria-label` that describes both the action to be taken and the current value (e.g., "Unlike post. Current likes: 5") because `aria-label` overrides the button's inner text. Apply `aria-hidden="true"` to any inner decorative icons. Provide a `title` attribute to explain *why* a button is disabled (e.g., "Sign in to like this post") or what it does when enabled.
