## 2026-04-08 - Added Keyboard Accessibility to Experience Timeline Cards
**Learning:** Custom interactive elements designed as cards (like the experience timeline accordions) often lack keyboard support (`tabIndex`, `role`, and `onKeyDown`) out of the box, rendering them unusable for keyboard and screen reader users. Simply adding an `onClick` handler to a `<div>` or custom UI card is insufficient for a11y.
**Action:** Always ensure that custom interactive elements acting as buttons have `role="button"`, `tabIndex={0}`, keyboard event handlers (for `Enter` and `Space`), and proper `aria-expanded`/`aria-controls` attributes if they toggle content. Ensure focus states are clear using `focus-visible`. Update click handler types (e.g. from `React.MouseEvent` to `React.SyntheticEvent`) to safely handle both mouse and keyboard inputs.## 2026-04-19 - Added Comprehensive ARIA Labels and State to Icon Buttons
**Learning:** Complex icon buttons like a 'Like' button with a counter require more than just an `aria-label`. Screen readers will announce both the label and the inner text content (the count). For state-driven buttons, it is essential to communicate the current state using `aria-pressed`.
**Action:** Always add `aria-pressed` to toggle buttons. Use a dynamic `aria-label` that describes both the action and the current value (e.g., 'Unlike post (5 likes)'), and hide the inner decorative icon AND text node from screen readers using `aria-hidden="true"` to prevent redundant reading. Also, provide a `title` attribute for mouse users to understand disabled states or actions.
