## 2026-04-08 - Added Keyboard Accessibility to Experience Timeline Cards
**Learning:** Custom interactive elements designed as cards (like the experience timeline accordions) often lack keyboard support (`tabIndex`, `role`, and `onKeyDown`) out of the box, rendering them unusable for keyboard and screen reader users. Simply adding an `onClick` handler to a `<div>` or custom UI card is insufficient for a11y.
**Action:** Always ensure that custom interactive elements acting as buttons have `role="button"`, `tabIndex={0}`, keyboard event handlers (for `Enter` and `Space`), and proper `aria-expanded`/`aria-controls` attributes if they toggle content. Ensure focus states are clear using `focus-visible`. Update click handler types (e.g. from `React.MouseEvent` to `React.SyntheticEvent`) to safely handle both mouse and keyboard inputs.## 2026-04-10 - State-Driven Icon Buttons Require Explicit ARIA Handling
**Learning:** State-driven icon buttons (like the Like button) often fail screen reader tests because `aria-label` overrides the inner text entirely. If the label doesn't dynamically reflect the state (e.g., "Like" vs "Unlike") or value (e.g., number of likes), screen reader users miss critical feedback.
**Action:** Always implement `aria-pressed` for toggle states, ensure the `aria-label` is dynamic (combining action and value), and use `aria-hidden="true"` on inner decorative icons to prevent redundant reading. Additionally, use `title` to explain why a button might be disabled (e.g., "Sign in to like").
