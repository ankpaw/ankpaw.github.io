## 2026-04-08 - Added Keyboard Accessibility to Experience Timeline Cards
**Learning:** Custom interactive elements designed as cards (like the experience timeline accordions) often lack keyboard support (`tabIndex`, `role`, and `onKeyDown`) out of the box, rendering them unusable for keyboard and screen reader users. Simply adding an `onClick` handler to a `<div>` or custom UI card is insufficient for a11y.
**Action:** Always ensure that custom interactive elements acting as buttons have `role="button"`, `tabIndex={0}`, keyboard event handlers (for `Enter` and `Space`), and proper `aria-expanded`/`aria-controls` attributes if they toggle content. Ensure focus states are clear using `focus-visible`. **Learning:** Icon-only or icon-dominant buttons that indicate state (like a 'Like' button) need comprehensive ARIA attributes. A visible number next to an icon is not sufficient for screen readers to understand the action.
**Action:** Always ensure state-driven buttons use `aria-pressed` to convey toggle state, combine the action and value in a clear `aria-label`, hide decorative icons with `aria-hidden='true'`, and explain disabled states with a `title` attribute.
