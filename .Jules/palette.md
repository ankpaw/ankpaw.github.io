## 2026-04-08 - Added Keyboard Accessibility to Experience Timeline Cards
**Learning:** Custom interactive elements designed as cards (like the experience timeline accordions) often lack keyboard support (`tabIndex`, `role`, and `onKeyDown`) out of the box, rendering them unusable for keyboard and screen reader users. Simply adding an `onClick` handler to a `<div>` or custom UI card is insufficient for a11y.
**Action:** Always ensure that custom interactive elements acting as buttons have `role="button"`, `tabIndex={0}`, keyboard event handlers (for `Enter` and `Space`), and proper `aria-expanded`/`aria-controls` attributes if they toggle content. Ensure focus states are clear using `focus-visible`. **Learning:** Adding just `aria-label` is not enough for stateful buttons (like Like/Unlike). `aria-pressed` should be used to indicate toggle state, and dynamic `aria-label` strings should include both the action and the value, while inner icons should be hidden using `aria-hidden="true"`. Tooltips (`title`) are particularly necessary for icon-only disabled states.
**Action:** Always pair `aria-pressed` with stateful icon buttons, hide inner SVG icons from screen readers, and consider tooltips (`title`) to explain disabled states.
