## 2026-04-08 - Added Keyboard Accessibility to Experience Timeline Cards
**Learning:** Custom interactive elements designed as cards (like the experience timeline accordions) often lack keyboard support (`tabIndex`, `role`, and `onKeyDown`) out of the box, rendering them unusable for keyboard and screen reader users. Simply adding an `onClick` handler to a `<div>` or custom UI card is insufficient for a11y.
**Action:** Always ensure that custom interactive elements acting as buttons have `role="button"`, `tabIndex={0}`, keyboard event handlers (for `Enter` and `Space`), and proper `aria-expanded`/`aria-controls` attributes if they toggle content. Ensure focus states are clear using `focus-visible`. Update click handler types (e.g. from `React.MouseEvent` to `React.SyntheticEvent`) to safely handle both mouse and keyboard inputs.## 2024-05-24 - Accessibility improvements for icon-only toggle buttons\n**Learning:** Icon-only toggle buttons (like "Like" buttons) need comprehensive accessibility attributes to be usable by screen readers. `aria-label` completely overrides the inner content for screen readers, so if you have a visible count next to the icon, the `aria-label` must include both the action and the count (e.g., `"Like post, 5 likes"`). Additionally, `aria-pressed` must be used to indicate toggle state, and decorative inner icons should be hidden with `aria-hidden="true"` to reduce noise.\n**Action:** When implementing or updating icon-only toggle buttons, always add `aria-pressed` for state, a comprehensive `aria-label` combining action and associated value, `aria-hidden="true"` on the inner SVG/icon, and consider a native `title` attribute for disabled states.
