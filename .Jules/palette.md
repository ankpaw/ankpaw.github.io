## 2026-04-08 - Added Keyboard Accessibility to Experience Timeline Cards
**Learning:** Custom interactive elements designed as cards (like the experience timeline accordions) often lack keyboard support (`tabIndex`, `role`, and `onKeyDown`) out of the box, rendering them unusable for keyboard and screen reader users. Simply adding an `onClick` handler to a `<div>` or custom UI card is insufficient for a11y.
**Action:** Always ensure that custom interactive elements acting as buttons have `role="button"`, `tabIndex={0}`, keyboard event handlers (for `Enter` and `Space`), and proper `aria-expanded`/`aria-controls` attributes if they toggle content. Ensure focus states are clear using `focus-visible`. Update click handler types (e.g. from `React.MouseEvent` to `React.SyntheticEvent`) to safely handle both mouse and keyboard inputs.
## 2026-05-05 - Added Comprehensive ARIA Labels to State-Driven Icon Buttons
**Learning:** State-driven icon buttons (like "Like" buttons or toggles) often need comprehensive `aria-label` attributes that combine both the action and the current value. This is because `aria-label` overrides the inner text (like a like count) for screen readers, leading to confusing reading experiences if only the action is described. Additionally, inner decorative icons (like hearts, spinners) need `aria-hidden="true"` so they do not add visual noise to screen reader output. Also, explicitly adding `title` attributes on buttons, particularly disabled ones, provides crucial context as to why an action is blocked.
**Action:** Always ensure that state-driven icon buttons combine action and value in `aria-label` (e.g. "Like this post, currently 12 likes"), add `aria-hidden="true"` to inner decorative SVG icons, and provide `title` attributes explaining why a button might be disabled (e.g. "Sign in to like this post").
