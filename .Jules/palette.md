## 2026-04-08 - Added Keyboard Accessibility to Experience Timeline Cards
**Learning:** Custom interactive elements designed as cards (like the experience timeline accordions) often lack keyboard support (`tabIndex`, `role`, and `onKeyDown`) out of the box, rendering them unusable for keyboard and screen reader users. Simply adding an `onClick` handler to a `<div>` or custom UI card is insufficient for a11y.
**Action:** Always ensure that custom interactive elements acting as buttons have `role="button"`, `tabIndex={0}`, keyboard event handlers (for `Enter` and `Space`), and proper `aria-expanded`/`aria-controls` attributes if they toggle content. Ensure focus states are clear using `focus-visible`. Update click handler types (e.g. from `React.MouseEvent` to `React.SyntheticEvent`) to safely handle both mouse and keyboard inputs.
## 2026-04-24 - Enhanced Accessibility and Tooltips for Interactive Icon Buttons
**Learning:** Icon buttons that track state (like a 'Like' button) and display a count alongside an icon can be confusing for screen reader users if the label doesn't specify the toggle state or action. Additionally, users may be confused when an interaction is disabled due to missing authentication without clear visual feedback.
**Action:** When implementing stateful icon buttons with accompanying text, use `aria-pressed` to denote toggle state, a dynamic `aria-label` to explicitly announce the action and context (e.g., 'Like this post. 5 likes'), and `aria-hidden='true'` on decorative inner icons. For disabled interactions requiring authentication, leverage the native `title` attribute to provide a simple, immediate tooltip explaining the required action.
