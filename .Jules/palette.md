## 2026-04-08 - Added Keyboard Accessibility to Experience Timeline Cards
**Learning:** Custom interactive elements designed as cards (like the experience timeline accordions) often lack keyboard support (`tabIndex`, `role`, and `onKeyDown`) out of the box, rendering them unusable for keyboard and screen reader users. Simply adding an `onClick` handler to a `<div>` or custom UI card is insufficient for a11y.
**Action:** Always ensure that custom interactive elements acting as buttons have `role="button"`, `tabIndex={0}`, keyboard event handlers (for `Enter` and `Space`), and proper `aria-expanded`/`aria-controls` attributes if they toggle content. Ensure focus states are clear using `focus-visible`. Update click handler types (e.g. from `React.MouseEvent` to `React.SyntheticEvent`) to safely handle both mouse and keyboard inputs.
## 2026-04-15 - ARIA Labels on Buttons with Text Content
**Learning:** When using `aria-label` on a button, it completely overrides the inner text content for screen readers. If a button's visual content includes important information (like a dynamic "Like count" number next to an icon), setting `aria-label="Like post"` will cause screen readers to completely miss the count number. Additionally, disabled icon buttons without visual explanations leave users confused about why an action is unavailable.
**Action:** Always incorporate the critical inner text values into the `aria-label` itself (e.g., `` aria-label={`Like post, currently ${count} likes`} ``), and then use `aria-hidden="true"` on the inner visual elements to prevent duplicate reading in some older assistive technologies. Always pair disabled states on icon-only/minimal buttons with a `title` explaining why it's blocked (e.g. `title="Sign in to like posts"`).
