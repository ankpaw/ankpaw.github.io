## 2026-04-08 - Added Keyboard Accessibility to Experience Timeline Cards
**Learning:** Custom interactive elements designed as cards (like the experience timeline accordions) often lack keyboard support (`tabIndex`, `role`, and `onKeyDown`) out of the box, rendering them unusable for keyboard and screen reader users. Simply adding an `onClick` handler to a `<div>` or custom UI card is insufficient for a11y.
**Action:** Always ensure that custom interactive elements acting as buttons have `role="button"`, `tabIndex={0}`, keyboard event handlers (for `Enter` and `Space`), and proper `aria-expanded`/`aria-controls` attributes if they toggle content. Ensure focus states are clear using `focus-visible`. **Learning:** Icon buttons with dynamically updating counters (like the like button) are often misread by screen readers if they only announce the number, missing the action context. Simply having inner text doesn't suffice if it lacks context or if an `aria-label` entirely overrides the value without including it. Furthermore, disabled states can confuse users if they lack a clear explanation (e.g. via a `title` attribute).
**Action:** When adding `aria-label` to buttons with dynamic values, ensure the label combines both the action context and the dynamic value (e.g., `aria-label="${hasLiked ? 'Unlike' : 'Like'} post, currently ${likes.size} likes"`). Add `aria-pressed` to toggle buttons. Use the `title` attribute on disabled buttons to explain why they are blocked (e.g., "Sign in to like this post"). Lastly, explicitly hide inner decorative icons with `aria-hidden="true"`.
